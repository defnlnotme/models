# Plan: Update vLLM and Test with Qwen3.5-9B on Intel

## Overview
Update the vLLM repository, build a Docker image for Intel XPU, test it, and fall back to a .venv setup with uv if Docker fails. Test with Qwen/Qwen3.5-9B model from Hugging Face.

## Steps

### 1. Update vLLM Repository
- Run `git fetch origin` to fetch latest changes
- Checkout `main` branch or latest release branch (releases/v0.17.1 or newer)
- Pull latest changes

### 2. Build Docker Image for Intel XPU
- Use `docker build` with `docker/Dockerfile.xpu`
- Build command: `cd /var/home/fra/dev/models/vllm-setup/vllm && docker build -f docker/Dockerfile.xpu -t vllm-xpu:latest .`
- The Dockerfile.xpu includes:
  - Base: intel/deep-learning-essentials:2025.3.2-0-devel-ubuntu24.04
  - Uses uv for Python package management
  - Targets XPU devices (Intel GPUs)
  - Sets VLLM_TARGET_DEVICE=xpu

### 3. Test Docker Image
- Update `/var/home/fra/dev/models/vllm-setup/vllm.sh` to use new image `vllm-xpu:latest`
- Update `/var/home/fra/dev/models/vllm-setup/vllm.yaml` to use model `Qwen/Qwen3.5-9B` with appropriate settings:
  - Reduce `tensor-parallel-size` to 1 (for single GPU)
  - Set `max-model-len` to 8192 or lower for testing
  - Use port 8009
- Run: `./vllm.sh` to start the server
- Test with curl request to `http://localhost:8009/v1/chat/completions`

### 4. Fallback: .venv Setup with uv (if Docker fails)
If Docker image doesn't work:

#### 4.1 Create Virtual Environment
```bash
cd /var/home/fra/dev/models/vllm-setup
uv venv .venv --python 3.12
source .venv/bin/activate
```

#### 4.2 Install vLLM with XPU Support
```bash
# Install Intel XPU specific requirements
uv pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/xpu
uv pip install -r vllm/requirements/xpu.txt
uv pip install vllm --no-build-isolation
```

Alternatively, if XPU build fails, install vLLM for CPU:
```bash
uv pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
uv pip install -r vllm/requirements/cpu.txt
VLLM_TARGET_DEVICE=cpu uv pip install vllm --no-build-isolation
```

#### 4.3 Run vLLM Server
```bash
export VLLM_TARGET_DEVICE=xpu  # or cpu if XPU not available
export ONEAPI_DEVICE_SELECTOR="level_zero:gpu"
export ZE_AFFINITY_MASK=0

vllm serve Qwen/Qwen3.5-9B --port 8009 --max-model-len 8192
```

### 5. Test with Qwen3.5-9B Model
- Model: `Qwen/Qwen3.5-9B` (Hugging Face)
- Test inference with a simple chat completion request
- Verify the model loads and generates responses
