# Plan: Patch build.sh to force GGML_NATIVE=ON for all backends

## Goal
Edit `build.sh` so that every Dockerfile used during `docker build` is temporarily patched to set `GGML_NATIVE=ON`, then cleaned up afterward — mirroring the existing temporary-patch pattern already used for `ik_llama_cpu`.

## Context
- Only `ik_llama_cpu` already defaults to `GGML_NATIVE=ON`. It stays unchanged.
- `intel.Dockerfile` has `GGML_NATIVE=OFF` hardcoded in the cmake line.
- `openvino.Dockerfile` and `vulkan.Dockerfile` mention `GGML_NATIVE` nowhere (CMake default `ON` applies implicitly, but only for build-time env, not inside the Dockerfile).
- The Dockerfiles do NOT expose `GGML_NATIVE` as a build ARG, so `docker build --build-arg GGML_NATIVE=ON` has no effect.

## Changes

### 1. Add `patch_for_native()` helper function (before `build_image()`)
A reusable function that:
- Takes a Dockerfile path
- Creates a mktemp copy
- Applies one of three sed patches based on what it finds:
  - `GGML_NATIVE=OFF` exists → `sed 's/GGML_NATIVE=OFF/GGML_NATIVE=ON/g'`
  - `GGML_NATIVE=ON` already present → no-op
  - No `GGML_NATIVE` at all → appends `-DGGML_NATIVE=ON` after the cmake command line
- Echos back the temp file path (or `""` on failure)
- Caller is responsible for cleanup

### 2. Update `build_image()` function
Replace the existing `ik_llama_cpu`-only temp-file block with a unified two-branch structure:

**Branch A — `ik_llama_cpu`:**
- GGML_NATIVE is already `= ON` in the source Dockerfile → no `patch_for_native` call
- Keep the existing llama-bench COPY sed-patch (temp file + `USE_TEMP_DOCKERFILE=1`)

**Branch B — all other backends (`openvino`, `intel`, `vulkan`):**
- Call `TEMP_DOCKERFILE=$(patch_for_native "$DOCKERFILE")`
- If the function returned a non-empty path, set `DOCKERFILE` to the temp file and set `USE_TEMP_DOCKERFILE=1`
- If the function returned `""` (file not found error), warn and continue with the original path

### 3. Cleanup
The existing cleanup block at the bottom of `build_image()` is unchanged and continues to handle both branches — it checks `USE_TEMP_DOCKERFILE` and removes the temp file.

## Backend coverage table

| Backend      | Pre-existing `GGML_NATIVE` | What patch_for_native does |
|---|---|---|
| `openvino`  | absent (nothing)         | append `-DGGML_NATIVE=ON` to cmake line |
| `intel`     | `= OFF`                  | replace `OFF` → `ON`                        |
| `vulkan`    | absent (nothing)         | append `-DGGML_NATIVE=ON` to cmake line |
| `ik_llama_cpu` | `= ON`               | skipped (already correct)                   |
